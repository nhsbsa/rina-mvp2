Principles - Automation

The work to integrate OHS RINA and OHS aims to reduce the amount of manual entry required by BSA agents to complete SEDs.

As part of the ongoing UCD work we are looking at ways to present auto-populated data, only where necessary, making this easy for the user to review, and if appropriate change. We are looking to avoid making users step through auto-populated forms in their entirety.

This guidance aims to help the development team create dynamic versions of the SEDs in a way that is accessible and consistent with the NHS design system.

However, it should be noted that in the cases we have so far explored as part of SBUC5 and SBUC1 - we do not expect users to be creating these forms from scratch, or working through them in their entirety.

For example, this is the current design thinking around how a user will create an S072 as part of the SBUC_1 workflow.

For discussion How might we create dynamic SEDs which only promote to the user the fields that need review or manual population? To be informed by the outputs of S1 testing.
