Principles - Routing

Routing is important to making the service simple to use and improving accessibility by reducing mental fatigue.

There are a number of redundant options in the existing SEDs which could be removed by using routing options.

The NHS service standard advises using filter questions to route users to answer only the questions that apply to them. These are not explicit in the current SEDs.

User research tells us that users rely on knowing what sections are required under different circumstances. We should look for opportunities to route users to the appropriate sections where possible.

What should we do?

Do not show redundant sections

We should only show SED sections that need to be checked or completed by the user, including where a section may or may not be required based on a previous answer.

Examples

S072: 5. INFORMATION ON ENTITLEMENT / 6. INFORMATION ON NO ENTITLEMENT

On an S072, for example, users know to only fill in section 5 or section 6 depending on whether the entitlement was issued or rejected.

We should use entitlement/no entitlement as a filter question routing users to the appropriate section only.

NOTE This may be achievable via automation directly from OHS, without the user having to manually indicate which decision was made.

S071: 4. IF YOU CHOOSE "A FAMILY MEMBER", PLEASE FILL IN SECTION INSURED PERSON

Use status of the person as a filter question - only proceed to section 4 if "A family member" is selected, otherwise proceed to section 5..

Do not show redundant fieldsets

There are certain sub-section headings, for example, IF PIN NOT PROVIDED FOR EVERY INSTITUTION, PLEASE PROVIDE that indicate they are only required based on a previous answer. If possible we should only show this fieldset if the described criteria is met by the previous answer.

The above list is not exhaustive and we acknowledge that it will need assessing for technical feasibility.

Automation may also play a role in routing and eliminating redundant entry.
